# 分布式备份系统

基于内容寻址的备份工具，通过 SHA256 哈希比较检测文件变化，实现单向安全同步。

## 工作原理

1. **扫描** 文件夹 —— 每个文件计算 SHA256 哈希，存入内容寻址对象库
   （`.backup/objects/ab/cdef...`），布局仿照 git 的对象存储。
2. **快照** —— 扫描结果（`相对路径 → 哈希` 的映射）自动保存为时间戳命名的
   JSON 快照（如 `20260627_113025`）。
3. **比较** 两个快照，得出四类差异：新增、修改、仅目标有、未变化。
4. **备份** —— 将新增和修改的文件从源复制到目标。目标侧独有的文件不会被删除。

备份前，GUI 会检查两项安全条件：文件夹是否在本次会话中重新扫描过，以及当前
快照是否是最新的。不满足时会弹窗警告，用户确认后才继续。

## 快速开始

```powershell
# 启动图形界面
python -m distbackup

# 或使用命令行
python -m distbackup cli scan D:\我的数据  笔记本快照
python -m distbackup cli scan D:\nas\备份  NAS快照
python -m distbackup cli diff  笔记本快照 NAS快照 --store D:\我的数据
python -m distbackup cli sync D:\我的数据 D:\nas\备份 笔记本快照 NAS快照
```

## 存储结构

```
源文件夹/
  .backup/
    objects/              # 内容寻址对象库（去重存储）
      0e/
        20aab55cbb...     # 文件内容按哈希命名存储
    snapshots/            # 时间戳命名的快照
      20260627_113025.json
      20260627_114201.json
```

每个快照记录：
- `created` — UTC 时间戳
- `root` — 被扫描目录的绝对路径
- `files` — `{相对路径: SHA256十六进制摘要}` 映射

## 图形界面操作流程

1. **Browse** 选择源文件夹和目标文件夹，自动加载各自最新的快照。
2. **Scan**（可选）重新扫描文件夹，自动保存新的时间戳快照。
3. 下拉框显示当前快照，`(new)` 标记表示最新快照。可从下拉列表选择历史快照加载。
4. 点击 **Compare** 查看差异树形列表。
5. 点击 **Start Backup** 执行同步。快照过期或未重新扫描时会弹出警告。

## 架构

| 模块 | 职责 |
|------|------|
| `ContentStore` | 管理 `.backup/objects/`，按 SHA256 哈希存取文件，自动去重。 |
| `Scanner` | 遍历目录树，对每个文件计算哈希，可选同步存入对象库。 |
| `SnapshotManager` | 保存和加载时间戳命名的 JSON 快照。 |
| `Differ` | 比较两个 `{路径: 哈希}` 映射，输出新增/修改/仅目标/未变化。 |
| `Syncer` | 将新增和修改的文件从源复制到目标，不删除目标中的旧文件。 |
