## 功能特性

### 扫描与哈希
- 递归遍历目录，基于 SHA256 内容寻址哈希（64 KB 分块流式读取）。
- 为根目录下每个文件生成 `{相对路径: SHA256摘要}` 映射。
- 自动跳过常见 VCS、工具链和操作系统目录：`.backup`、`.git`、`.svn`、`.hg`、`__pycache__`、`.venv`、`venv`、`node_modules`、`.idea`、`.vscode`、`target`、`vendor` 等。
- 遇到无法读取的文件时记录日志并跳过，而非中断扫描。

### 版本化快照
- 时间戳命名的 JSON 快照（如 `20260627_113025.json`），存储于 `.backup/snapshots/`。
- 原子写入（临时文件 + `os.replace`），崩溃不会留下残缺的 JSON。
- 每条快照记录扫描时间（UTC ISO 格式）、扫描根目录的绝对路径、以及完整的文件到哈希映射。
- 快照根目录校验，防止将一个目录的快照用于另一个目录。

### 差异引擎
- 静态比较两个 `{路径: 哈希}` 映射，输出四类结果：新增、修改、仅目标有、未变化。
- 轻量 `DiffResult` 数据类，附带 `total_changes` 便捷属性。

### 单向同步（源 -> 目标）
- 仅复制新增和修改的文件到目标 -- 目标端独有的文件不会被删除。
- 路径穿越防护：每个目标路径均被验证为位于目标目录内部。
- `--dry-run` 模式可预览将要复制的文件，不执行实际写入。
- 逐文件错误追踪，单个失败不影响其余复制任务。
- 复制前交互式确认（可通过 `--force` 跳过）。

### 备份后验证
- 每次同步后自动重新扫描目标目录。
- 将目标的最新扫描结果与源快照比对以确认正确性。
- 验证结果保存为新的时间戳目标快照。

### 仓库类型保护
- 每个目录在 `.backup/config.json` 中记录类型：`Source`（源）或 `Target`（目标，默认值）。
- 标记为 `Source` 的目录受写保护 -- sync 操作拒绝向其写入。
- 在 GUI 中从 `Source` 切换到 `Target` 会弹出确认对话框。
- 类型可通过 CLI 和 GUI 查看及设置。

### 存储结构
- 所有元数据存放于隐藏的 `.backup/` 目录下（Windows 上通过 `SetFileAttributesW` 自动隐藏）。
- `.backup/config.json` 保存仓库类型。
- `.backup/snapshots/` 保存时间戳命名的 JSON 快照文件。

### CLI
- `scan <目录> <名称>` -- 对目录做哈希扫描并保存快照。
- `diff <源快照> <目标快照>` -- 比较两个快照，输出可读差异。
- `sync <源目录> <目标目录> <源快照> <目标快照>` -- 复制变更文件，支持 `--dry-run` 和 `--force`。
- `list` -- 列出所有快照，显示文件数和时间戳。
- `type <目录> [Source|Target]` -- 读取或设置仓库类型。

### GUI（tkinter）
- 浏览选择源和目标文件夹，自动加载最新快照。
- 只读快照下拉框，最新条目标记 `(new)`。
- 双侧扫描按钮，后台线程执行并反馈进度。
- 一键 Compare，在树形视图中展示新增、修改、仅目标有的文件。
- 备份前安全检查：快照过期警告、文件夹未扫描警告、可选确认对话框。
- 备份后验证结果在同一个差异树中以 OK / FAILED 标记展示。
- Toggle 按钮切换仓库类型，关键安全转换需确认。
- 键盘快捷键：`Ctrl+O` 浏览源、`Ctrl+Shift+O` 浏览目标、`Ctrl+R` 刷新快照、`Ctrl+W` / `Ctrl+Q` 退出。

### 测试套件
- 107 个测试，覆盖哈希、扫描、差异、同步逻辑、快照管理、CLI 命令、GUI 静态结构、GUI 逻辑方法及端到端集成。
- 使用隔离的 `tmp_path` 夹具，无测试触及真实用户目录。

### 依赖
- 仅需 Python 3.10+ 标准库：`hashlib`、`os`、`shutil`、`json`、`tkinter`、`argparse`、`tempfile`、`ctypes`、`threading`、`dataclasses`。
